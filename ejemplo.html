Ejercicio

Subida Insegura de Archivos


El siguiente código permite cargar archivos sin validar el nombre, extensión o contenido, lo que puede representar un riesgo de seguridad. 



String fileName = request.getParameter("fileName");

File file = new File("/uploads/" + fileName);
FileOutputStream fos = new FileOutputStream(file);

fos.write(request.getInputStream().readAllBytes());
fos.close();

Problemas a Solucionar


Problema ¿Por qué es un problema? Ejemplo de ataque
1 Path Traversal El parámetro fileName se concatena directamente a la ruta /uploads/. Un atacante puede usar ../ para salir del directorio permitido. fileName = "../../etc/passwd" → accede a archivos del sistema.
2 Sin validación de extensión No se verifica que el archivo subido sea de un tipo permitido (ej: imagen, PDF). fileName = "virus.exe" → se sube un ejecutable malicioso.
3 Sobrescritura arbitraria FileOutputStream(file) no verifica si el archivo ya existe. Sobrescribe cualquier archivo sin preguntar. fileName = "configuracion.properties" → destruye configuración crítica.
4 Denegación de servicio (DoS) readAllBytes() carga todo el archivo en memoria sin límite de tamaño. Enviar un archivo de 10 GB → colapsa la memoria del servidor.
5 Sin control de contenido real Solo confía en el nombre del archivo, no verifica el contenido (magic bytes). foto.jpg pero realmente es un script malicioso.
6 Sin manejo de errores Si algo falla (disco lleno, ruta inválida), no hay control de excepciones. El servidor puede mostrar información interna en mensajes de error.
